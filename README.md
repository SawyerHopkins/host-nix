# Host Nix

Provides `dot_files` extension to [Host Init](https://github.com/SawyerHopkins/host-init). Sets up a `~/Code` directory with podman and the following commands to enable developing and running projects inside containerized environments.

| Script         | Description                                                                                                                                                                                                         | Usage                                                                                |
|----------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------|
| pm-up          | Starts a podman container with the current directory mounted for editing. Container name defaults to current directory name. Defaults to the [Podman TS Dev](https://github.com/SawyerHopkins/podman-ts-dev) image. | pm-up container_name=$(pwd) image=https://github.com/SawyerHopkins/podman-ts-dev.git |
| pm-down        | Stops a podman container. By default will try to stop the container with the same name as the current directory.                                                                                                    | pm-down container_name=$(pwd)                                                        |
| pm-destory     | Remove a podman container. By default will try to stop the container with the same name as the current directory.                                                                                                   | pm-destory container_name=$(pwd)                                                     |
| pm-ssh-tunnel  | Meta command that makes SSH agent available for forwarding into podman containers.                                                                                                                                  | pm-ssh-tunnel                                                                        |
| pm-ssh-kill    | Closes socket used for SSH agent forwarding.                                                                                                                                                                        | pm-ssh-kill                                                                          |
| pm-ssh-chmod   | Fixes socket file permissions to allow SSH agent forwarding from VM to container. Ran by `pm-ssh-tunnel` automatically.                                                                                             | pm-ssh-chmod                                                                         |
| pm-ssh-forward | Opens a SSH socket using -R -N to forward socket into podman VM. Ran by `pm-ssh-tunnel` automatically.                                                                                                              | pm-ssh-forward                                                                       |
| pm-ssh-clean   | Removes any old/dead forwarding socket in the VM. Ran by `pm-ssh-tunnel` automatically                                                                                                                              | pm-ssh-clean                                                                         |
